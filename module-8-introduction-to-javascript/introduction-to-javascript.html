‚ö° What is JavaScript?

üåê
The Language of the Web

JavaScript runs in browsers and adds interactivity. HTML (structure) + CSS (style) + JavaScript (behavior).

üöÄ
Powers Modern Web

Form validation, animations, real-time updates, dynamic content, games, full apps. The brain of web pages.

üéØ
Versatile

Browser (client-side), server (Node.js), desktop apps, mobile. Everywhere you need dynamic behavior.

üìö
Easy to Start, Hard to Master

Basics are accessible. Advanced patterns take time. Millions of libraries and frameworks available.

üì¶ Variables & Data Types

üéÅ
Variables: Containers for Data

Use let or const. camelCase naming. Descriptive names matter.

üìù
Data Types

String: "text", Number: 42, Boolean: true/false, Array: [1,2,3], Object: {name: "Alice"}, undefined, null

‚öñÔ∏è
const vs let

Use const by default (prevents reassignment). Use let only when you need to reassign.

üîç
Type Checking

Use typeof to check type: typeof "hello" ‚Üí "string". Important for understanding values.

Code Examples

// Strings
const name = "Alice";
const message = "Hello";

// Numbers
let age = 25;

// Arrays

  ["apple", "banana"];

// Objects
const person = {
  name: "Bob",
  age: 30,
  city: "NYC"
};

// Booleans
const isStudent = true;

üî¢ Operators & Control Flow

‚ûï
Operators

Arithmetic: +, -, *, /, %. Comparison: ===, !==, <, >. Logical: &&, ||, !

ü§î
if/else: Making Decisions

Test conditions and execute different code paths. Use && and || to combine conditions.

üîÑ
Loops: Repeating Code

for, while, for...of. Execute code multiple times efficiently.

‚öôÔ∏è
Use === not ==

Strict equality (===) compares type and value. Avoid loose equality (==) which coerces types.

Code Examples

control-flow.js 
// if/else
if (age >= 18) {
  console.log("Adult");
} else {
  console.log("Minor");
}

// for loop
for (let i = 0; i < 5; i++) {
  console.log(i);
}

// while loop
let count = 0;
while (count < 3) {
  console.log(count);
  count++;
}

// Logical operators
if (age > 18 &&
    hasLicense) {
  console.log("Can drive");
}

Output

if/else Result:
"Adult"
for loop Output:
0 1 2 3 4
while loop Output:

1

üîß Functions: Reusable Code

üìã
What is a Function?

A named block of code that performs a task. Takes input (parameters), returns output (return value).

üéØ
Declaration vs Calling

function greet(name) { } declares. greet("Alice") calls/executes it.

‚Ü©Ô∏è
Return Values

return sends value back to caller. Without return, function returns undefined.

‚ö°
Arrow Functions

Modern syntax: const add = (a, b) => a + b; is shorter than traditional function.

Code Examples

functions.js 
// Declaration
function add(a, b) {
  return a + b;
}

// Calling
let result = add(5, 3);

// Arrow function
const multiply =
  (a, b) => a * b;

// With default
function greet(
  name = "Friend"
) {
  return `Hello,
          ${name}!`;
}

// Calling functions
add(10, 20);
greet("Alice");

Results

‚Üí 8
multiply(4, 5)
‚Üí 20
greet("Bob")
‚Üí "Hello, Bob!"

üìä Arrays & Objects: Organizing Data

üìã
Arrays: Collections

Store multiple values. Zero-indexed: first element is [0]. Access with array[0].

üîÑ
Array Methods

push(), pop(), map(), filter(), forEach(). Super powerful for data manipulation.

üóùÔ∏è
Objects: Key-Value Pairs

Organize related data: {name: "Alice", age: 25}. Access with dot or bracket notation.

üéØ
When to Use

Arrays for lists. Objects for entities. Arrays of objects for collections of entities (users, products).

Code Examples

arrays-objects.js 
// Array
const numbers =
  [1, 2, 3, 4, 5];
numbers.push(6);

// Array methods
const doubled =
  numbers.map(n => n * 2);
const evens =
  numbers.filter(n =>
    n % 2 === 0);

// Object
const person = {
  name: "Alice",
  age: 25,
  city: "NYC"
};
person.age = 26;

// Array of objects
const users = [
  {name: "Alice",
   age: 25},
  {name: "Bob",
   age: 30}
];

üéÆ Practical JavaScript

üìù
Console Logging

console.log() prints to browser console. Essential for debugging. Use for testing variables.

üêõ
Debugging

Browser DevTools (F12) shows errors. Use console.log strategically. Check variable values step by step.

‚úÖ
Best Practices

Use meaningful names. Keep functions small. Comment confusing logic. Test frequently. Handle errors gracefully.

‚ö°
Next Level: DOM Interaction

Common Patterns

‚úÖ Good:
  • Descriptive variable names
  • Small focused functions
  • Use const by default
  • Comments for why, not what
‚ùå Avoid:
  • Single-letter variables
  • Functions that do too much
  • Global variables
  • Silent errors

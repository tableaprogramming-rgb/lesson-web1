üå≥ The Document Object Model (DOM)

üèóÔ∏è
DOM is a Tree

Browser converts HTML into a tree structure. Each element becomes an object you can access and manipulate.

üì¶
Everything is an Object

HTML elements, attributes, text are all objects. Access via JavaScript variables and properties.

üîó
Relationships Matter

Parent, child, sibling relationships. Navigate tree: element.parentElement, element.children.

‚ö°
Live Connection

Changes via JavaScript update page instantly. Modify one element, entire page reflects change.

üîç Selecting Elements: Finding What to Modify

üéØ
getElementById

document.getElementById("myId") selects element by ID. Fast, single element.

üîé
querySelector

document.querySelector(".myClass") uses CSS selectors. Modern, recommended approach.

üìã
querySelectorAll

document.querySelectorAll("p") selects multiple elements. Returns NodeList (like array).

üí°
Best Practice

Use querySelector/querySelectorAll for consistency. They support all CSS selectors.

Selection Code

selection.js 
// By ID
const header =
  document.getElementById(
    "header"
  );

// By class (first only)
const btn =
  document.querySelector(
    ".primary-btn"
  );

// All paragraphs
const paras =
  document.querySelectorAll(
    "p"
  );

// Complex selector
const content =
  document.querySelector(
    "main .article p"
  );

What It Selects

getElementById
Single element or null
querySelector
First match or null
querySelectorAll
All matches (NodeList)

‚úèÔ∏è Manipulating Content & Styles

üìù
textContent

Get/set plain text. Safe, no HTML parsing. Use for text-only content.

üè∑Ô∏è
innerHTML

Get/set HTML (including tags). Powerful but be careful with user input (XSS risk).

üé®
classList for Classes

element.classList.add("active"), .remove(), .toggle(). Preferred over inline styles.

üíª
style Property

element.style.color = "red". Direct inline styling. Use classList instead when possible.

Manipulation Code

manipulation.js 
// Change text
para.textContent =
  "New text";

// Change HTML
div.innerHTML =
  "<strong>Bold</strong>";

// Add class
btn.classList.add(
  "active"
);

// Toggle class
element.classList
  .toggle("hidden");

// Inline style
element.style.backgroundColor
  = "blue";

üñ±Ô∏è Event Handling: Responding to Users

üéØ
What are Events?

Things that happen: click, submit, keydown, mouseover. JavaScript responds with event listeners.

üëÇ
addEventListener

element.addEventListener("click", function). Modern way. Multiple listeners supported.

üìã
Common Events

click, submit, input, keydown, keyup, change, mouseover, mouseout, focus, blur

üö´
Prevent Default

event.preventDefault() stops default behavior (form submission, link navigation).

Event Code

events.js 
// Click event
btn.addEventListener(
  "click",
  () => {
    console.log("Clicked!");
  }
);

// Form submit
form.addEventListener(
  "submit",
  (e) => {
    e.preventDefault();
    console.log("Submitted");
  }
);

// Input typing
input.addEventListener(
  "input",
  (e) => {
    console.log(e.target
      .value);
  }
);

üÜï Creating & Removing Elements Dynamically

üèóÔ∏è
createElement

document.createElement("div") creates new element. Not in DOM yet‚Äîmust append.

üìç
appendChild & insertBefore

parent.appendChild(child) adds element. insertBefore adds before specific element.

üóëÔ∏è
remove

element.remove() deletes from DOM. Or parent.removeChild(child).

üí°
Real Use Case

Todo lists, dynamic lists, components. Build UIs that respond to user actions.

Creation Code

create-elements.js 
// Create element
const li =
  document.createElement(
    "li"
  );

// Set content
li.textContent =
  "New item";

// Add class
li.classList.add(
  "item"
);

// Add to page
const list =
  document.querySelector(
    "ul"
  );
list.appendChild(li);

// Remove
li.remove();

üéØ Event Delegation & Best Practices

üîÑ
Event Delegation

Listen on parent for events from children. Perfect for dynamic elements added later.

üåä
Event Bubbling

Events bubble up from child to parent. Use event.target to see which element triggered.

‚ö°
Performance

Delegate instead of adding listeners to each element. One listener on parent > many on children.

üí°
Practical Tips

Store references to elements. Use data attributes for info. Keep event handlers small and focused.

Best Practices Summary

‚úÖ Do:
  • Use addEventListener
  • Use event delegation
  • Cache DOM references
  • Use classList for styles
  • Handle errors gracefully
‚ùå Don't:
  • Use inline event handlers
  • Query DOM repeatedly
  • Modify innerHTML with user input
  • Use inline styles (use classes)
  • Ignore performance

üé® What is CSS?

üí°
The Cascade

Rules flow top-to-bottom. If two rules have equal "weight", the last one defined wins.

üå≥
Inheritance

Properties like color and font-family pass from parent to child. border and margin do NOT.

üìÇ
Specificity

A weighting system (Element, Class, ID) that determines which rule is objectively "stronger".

üåä
Cascade Layers (@layer)

A modern way to organize styles into prioritized groups regardless of selector specificity.

HTML Code

style.html 
<head>
  <link rel="stylesheet"
        href="styles.css">
  <style>
    p { color: navy; }
  </style>
</head>
<body>
  <h1>Hello</h1>
  <p style="color: red;">
    Inline style
  </p>
</body>

‚öñÔ∏è Specificity & Cascade

üèÜ
Specificity Points

Element = 1pt, Class = 10pts, ID = 100pts, Inline = 1000pts. Higher specificity wins.

üìä
Calculating Specificity

.article p { } = 11pts (1 class + 1 element). #main p { } = 101pts (1 ID + 1 element).

üè∑Ô∏è
Class vs ID

Classes (.) are for reusable styles (many elements). IDs (#) are unique (one element per page) and carry more weight.

‚ö†Ô∏è
Best Practices

Favor Classes for flexibility. Use IDs for unique JS hooks or jump-links, not primary styling.

üåä
Cascade

CSS rules flow top-to-bottom. Later rules override earlier ones *if specificity is equal*.

Example Code

specificity.css 
/* 1 point */
p { color: black; }

/* 11 points - WINS */
.article p {
  color: navy;
}

/* 101 points - HIGHEST */
#main p {
  color: darkblue;
}

/* 1000 points - BAD! */
p {
  color: red !important;
}

üìè Modern Sizing: Beyond Pixels

üè†
Relative Units (rem/em)

rem: Relative to root (html) font size. em: Relative to parent/element size. Scale safely!

üñ•Ô∏è
Viewport Units (vh/vw)

Size elements relative to the screen width/height. Perfect for full-screen hero sections.

üìê
Logical Properties

Use margin-inline instead of left/right to support various writing directions (RTL).

üì¶
border-box

Always use box-sizing: border-box to include padding and border in the total size.

Logical vs Physical

/* Physical: always left */
.old { margin-left: 20px; }

/* Logical: adapts to text flow */
/* (e.g. margin-right in RTL) */
.modern { margin-inline-start: 20px; }
Physical: Static position (Hardcoded Left)
Logical: Dynamic (Flow-relative Start)

üõ†Ô∏è Power Tools: Variables & Functions

üé®
Custom Properties (Variables)

Store values once and reuse globally: --primary-color: #6366f1;

üßÆ
calc()

Perform math in CSS: width: calc(100% - 20px);

üóúÔ∏è
clamp(MIN, PREF, MAX)

Responsive sizing without media queries. Example: clamp(1rem, 5vw, 3rem) stays between 1-3rem.

‚öñÔ∏è
min() & max()

Provide lower and upper bounds for element dimensions.

Modern Syntax

:root {
  --primary: #818cf8;
}

h1 {
  color: var(--primary);
  font-size: clamp(2rem, 10vw, 5rem);
  width: min(90%, 1200px);
}

üì¶ The Box Model

üéÅ
Four Layers

Content (innermost) ‚Üí Padding ‚Üí Border ‚Üí Margin (outermost). Every HTML element is a box.

üìê
Margin

Space *outside* the border (transparent). Creates distance between elements.

üìè
Padding

Space *inside* the border. Creates breathing room for content.

‚öôÔ∏è
box-sizing: border-box

Include padding/border in width/height for predictable sizing. Apply globally with * { }.

CSS Code

box-model.css 
* {
  box-sizing: border-box;
}

.box {
  width: 200px;
  margin: 20px;
  border: 2px solid
          #999;
  padding: 15px;
  background:
    lightblue;
}

üé® Colors & Typography

üåà
Color Formats

Hex #FF5733, RGB rgb(255, 87, 51), HSL hsl(9, 100%, 64%), or named colors.

üî§
Font Family

System fonts (Arial) or web fonts (Google Fonts). Always provide fallback chain: 'Inter', sans-serif.

üìè
Font Properties

font-size, font-weight, line-height. Use 16px baseline, 1.6 line-height for readability.

üñºÔ∏è
Backgrounds

Solid color, image, or gradient. Use background-size: cover, background-position: center.

üñºÔ∏è Backgrounds & Visual Effects

üé®
Background Properties

background-color, background-image, background-size, background-position, background-repeat.

üìê
Background Size

cover fills area (may crop), contain fits entire image, or specific pixel values.

üåà
Gradients

Linear: linear-gradient(90deg, color1, color2). Radial: radial-gradient(circle, ...). No image needed.

üí°
Performance Tips

Optimize images (compress, use appropriate format). Gradients are lightweight. Use appropriate file sizes.

CSS Code

backgrounds.css 
/* Solid color */
.box1 {
  background-color:
    #F0F0F0;
}

/* Linear gradient */
.box2 {
  background:
    linear-gradient(
      135deg,
      #667eea 0%,
      #764ba2 100%
    );
}

/* Radial gradient */
.box3 {
  background:
    radial-gradient(
      circle,
      yellow,
      purple
    );
}

Element Selector

Element selectors target HTML elements by their tag name. In this example, the CSS rule for the p tag applies styling to every paragraph on the page. Element selectors are the most basic type of selector and are perfect for applying consistent styling to all instances of a particular element.

Specificity: 0, 0, 1

ID Selector

ID selectors target a unique element using the hash symbol (#) followed by the element's id attribute. In this example, the paragraph has id="second", which can be styled with #second { /* styles */ }. IDs must be unique within an HTML documentâ€”no two elements should share the same ID.

Specificity: 1, 0, 0

Class Selector

Class selectors target elements using the dot (.) followed by the class name. Multiple elements can share the same class, making class selectors ideal for reusing styling across different parts of your page. Notice the highlight span element also has a class.

Specificity: 0, 1, 0

CSS Cascade

The cascade is the fundamental principle that determines which CSS rules are applied when multiple rules target the same element. When conflicts arise, CSS follows a specific order of priority. Rules that appear later in the stylesheet override earlier rules with the same specificity.

Order of Appearance matters

CSS Specificity

Specificity is a measure of how precisely a CSS selector targets an element. If you have p { color: blue; } and .red { color: red; }, the class selector wins because it has higher specificity.

Weighted Priority: ID > Class > Element
